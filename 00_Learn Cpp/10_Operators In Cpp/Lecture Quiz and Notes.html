Precedence and Associativity

üìä Understanding Operator Precedence
üîë Key Concepts:
  • Precedence: Operators have a defined precedence which dictates the order of operation in expressions
  • Associativity: Determines the order of operations with operators of the same precedence, typically left-to-right or right-to-left
üìù Example

In the expression 3 + 4 * 5, multiplication has a higher precedence, so 4 * 5 is evaluated first.

Result: 3 + 20 = 23

Examples and Dry Run

üíª Illustrative Example 
int a = 5, b = 10;
cout << (a + b * 2); // Outputs: 25 due to precedence of * over +

Explanation: The multiplication operator * has higher precedence than the addition operator +. Therefore, b * 2 is evaluated first (10 * 2 = 20), then added to a (5 + 20 = 25).

Common Mistakes with Loops

‚ùå Infinite Loops

Using incorrect loop conditions can lead to infinite loops.

// Wrong: Creates infinite loop
int i = 0;
while (i < 10) {
    cout << i << endl;
    // No increment -> Infinite loop
}

// Correct: Proper increment
int i = 0;
while (i < 10) {
    cout << i << endl;
    i++;  // Don't forget this!
}
‚ùå Misplaced Increment Operators

Misunderstanding where to place increment operators in for and while loops can lead to logic errors.

// Be careful with placement
for (int i = 0; i < 5; i++) {
    cout << i << endl;  // Correct placement
}

// vs
for (int i = 0; i < 5;) {
    cout << i << endl;
    i++;  // Also correct, but less common
}
