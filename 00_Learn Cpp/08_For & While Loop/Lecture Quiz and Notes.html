For and While Loops in C++

Mastering Repetitive Control Structures

Overview: In this lecture, we delve deeply into the concepts of For and While Loops in C++. These are essential control flow structures that allow repeated execution of a block of code. The session outlines the essential components and the mechanics of each loop, common pitfalls, and practical code examples to ground the theory.

For Loop

üîÑ For Loop

A for loop is typically used when the number of iterations is known beforehand. It consists of three parts:

1 Initialization:

Sets up a variable for iteration

2 Condition:

Checks a boolean expression to determine if the loop should continue

3 Iteration Expression:

Updates the iterator after each loop cycle

üíª For Loop Example 
for (int i = 0; i < 5; i++) {
    std::cout << "Iteration " << i << std::endl;
}

Breakdown:

  • int i = 0 - Initialization: i is set to 0
  • i < 5 - Condition: Loop continues as long as i is less than 5
  • i++ - Iteration: i is incremented after each iteration

While Loop

üîÅ While Loop

A while loop repeats a block of code as long as a given condition remains true. It is suited for situations where the number of iterations is not known ahead of time.

üíª While Loop Example 
int count = 0;
while (count < 5) {
    std::cout << "Iteration " << count << std::endl;
    count++;
}

How it works: This loop uses count as a control variable and runs until count becomes greater than or equal to 5.

üí° Key Difference:

For Loop: Best when you know the number of iterations
While Loop: Best when iterations depend on a condition that may change unpredictably

Infinite Loop

‚ö†Ô∏è Infinite Loop Warning

Both for and while loops can run indefinitely if the termination condition is never met.

üíª Example of Infinite While Loop 
while (true) {
    // This loop will run forever
}
‚úÖ Solution:

To break out of such loops, use a break statement based on specific conditions.

while (true) {
    if (someCondition) {
        break;  // Exit the loop
    }
}

Common Mistakes

Be aware of these common pitfalls when working with loops:

‚ùå 1. Off-by-one Errors

Incorrect initialization or condition leading to one extra or one less iteration.

// Wrong: Runs 6 times instead of 5
for (int i = 0; i <= 5; i++) { }

// Correct: Runs exactly 5 times
for (int i = 0; i < 5; i++) { }
‚ùå 2. Forgotten Update

In for loops, failing to update the iterator variable can lead to infinite loops.

// Wrong: Infinite loop (no increment)
for (int i = 0; i < 5;) {
    std::cout << i;
}

// Correct: Proper increment
for (int i = 0; i < 5; i++) {
    std::cout << i;
}
‚ùå 3. Logical Errors in Conditions

Conditions that never resolve to false can freeze execution.

// Wrong: Condition never becomes false
int i = 0;
while (i < 10) {
    std::cout << i;
    // Forgot to increment i!
}

// Correct: Proper update
int i = 0;
while (i < 10) {
    std::cout << i;
    i++;  // Don't forget this!
}
