Arguments vs Parameters

üî§ Understanding the Difference
üìå Parameters:

Variables defined in the function definition are called parameters. They act as placeholders for the values that are passed during the function call.

üìå Arguments:

The actual values that you pass to the function when you call it are known as arguments.

üíª Example to Illustrate 
int multiply(int x, int y) {  // x and y are parameters
    return x * y;
}

int main() {
    int result = multiply(4, 5);  // 4 and 5 are arguments
    cout << "Product: " << result << endl;
    return 0;
}
üìä Breakdown
  • int x, int y in the function definition are parameters
  • 4, 5 passed in multiply(4, 5) are arguments
  • Parameters are placeholders; arguments are actual values

Call by Value vs Call by Reference

Understanding these concepts is crucial for manipulating data efficiently:

üì• Call by Value

Call by Value: Copies the actual value of an argument into the formal parameter of the function. Changes made to the parameter inside the function have no effect on the argument.

üíª Call by Value Example 
void modifyValue(int a) {
    a = 100; // Changes made here won't affect the original
}

int main() {
    int num = 20;
    modifyValue(num);
    cout << "Num after modifyValue: " << num << endl; // Output: 20
    return 0;
}

Output: Num after modifyValue: 20

Explanation: The value of num remains 20 because only a copy was passed to the function.

üîë Key Characteristics:
  • A copy of the actual parameter's value is created
  • Changes to the parameter do not affect the original argument
  • Original data is safe from modification
  • Less efficient for large data structures
üîó Call by Reference

Call by Reference: Copies the reference of an argument into the formal parameter. The function works with the original argument, and changes will reflect on the argument used in the call.

üíª Call by Reference Example 
void modifyReference(int &b) {
    b = 100; // Changes here affect the original argument
}

int main() {
    int num = 20;
    modifyReference(num);
    cout << "Num after modifyReference: " << num << endl; // Output: 100
    return 0;
}

Output: Num after modifyReference: 100

Explanation: The value of num is changed to 100 because the function received a reference to the original variable.

üîë Key Characteristics:
  • The parameter becomes an alias for the argument
  • Changes to the parameter affect the original argument
  • More efficient for large data structures
  • Denoted by the & symbol in parameter declaration
‚öñÔ∏è Call by Value vs Call by Reference
Call by Value:
  • Passes a copy of the value
  • Original data remains unchanged
  • Safer but less efficient for large data
  • Example: void func(int x)
Call by Reference:
  • Passes a reference to the value
  • Original data can be modified
  • More efficient for large data structures
  • Example: void func(int &x)
üí° Pro Tip:

Use call by value when you want to protect the original data from being modified. Use call by reference when you need to modify the original data or when working with large data structures to improve efficiency.
