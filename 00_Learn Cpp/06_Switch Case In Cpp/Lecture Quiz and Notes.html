Switch Case Statement in C++

Efficient Decision Making with Multiple Options

Overview: In this lecture, we explore the Switch Case statement in C++, a conditional control structure that is commonly used for decision-making where a variable is compared against a list of values. Switch statements help in writing cleaner and more efficient code when compared with multiple if-else conditions.

Key Aspects of Switch Case

üìã Syntax and Usage

A switch statement evaluates a single expression, which is usually an integer or character, then executes the block of code associated with the matching case. If no case matches, the control can optionally transfer to the default case.

üíª Basic Switch Syntax 
switch (expression) {
    case constant1:
        // code block
        break;
    case constant2:
        // code block
        break;
    // additional cases
    default:
        // default code block
}
üî§ Valid Data Types

The expression in a switch statement must be of an integral or enumerated data type. Floating point values and other complex data types are not supported directly.

Supported Types:

int char enum short long
‚ùå Not Supported:

float, double, string, and other complex data types cannot be used directly in switch statements.

Best Practices

‚úÖ Always Use Break Statements

Include a break statement in each case to prevent fall-through, unless intentionally allowing multiple cases to execute the same block of code.

break; prevents execution from continuing to the next case
‚úÖ Include a Default Case

Use the default case to handle unexpected input that doesn't match any case. This ensures your program handles all possible inputs gracefully.

default: acts as a catch-all for unmatched values

Important Points

‚ö° Efficiency: Switch statements offer an efficient way to branch code execution compared to multiple if-else constructs when dealing with a single integer or character condition.

üì¶ Variable Scope: Variables declared within a case exist only within that block unless declared outside the switch statement.

üîÄ Fall-Through Behavior: Without a break statement, execution will "fall through" to the next case, which can be intentional or accidental.

Code Example

Consider a program that outputs the day of the week based on a number input:

üíª Complete Day of Week Program 
#include <iostream>
using namespace std;

int main() {
    int day;
    cout << "Enter day number (1-7): ";
    cin >> day;

    switch (day) {
        case 1:
            cout << "Monday" << endl;
            break;
        case 2:
            cout << "Tuesday" << endl;
            break;
        case 3:
            cout << "Wednesday" << endl;
            break;
        case 4:
            cout << "Thursday" << endl;
            break;
        case 5:
            cout << "Friday" << endl;
            break;
        case 6:
            cout << "Saturday" << endl;
            break;
        case 7:
            cout << "Sunday" << endl;
            break;
        default:
            cout << "Invalid day!" << endl;
    }

    return 0;
}

üí° How It Works: Using the code above, the program evaluates the day input and executes the corresponding case block. If a number outside 1-7 is entered, the default case outputs "Invalid day!"

üéØ Example Execution:
  • Input: 3 ‚Üí Output: Wednesday
  • Input: 7 ‚Üí Output: Sunday
  • Input: 9 ‚Üí Output: Invalid day!
